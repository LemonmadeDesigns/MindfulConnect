# Contributing to MindfulConnect

Thank you for your interest in contributing to MindfulConnect! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and inclusive
- Keep discussions focused and professional
- Handle sensitive data with care
- Follow HIPAA compliance guidelines
- Report security concerns immediately

## How to Contribute

### Setting Up Development Environment

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/mindfulconnect.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `pip install -r backend/requirements.txt`

### Development Guidelines

1. Follow PEP 8 style guide
2. Write clear, commented code
3. Include docstrings for functions and classes
4. Add type hints where applicable
5. Write unit tests for new features

### Pull Request Process

1. Update documentation if needed
2. Add tests for new features
3. Ensure all tests pass
4. Update requirements.txt if new dependencies are added
5. Create a detailed PR description

### Commit Message Guidelines

Follow conventional commits format:

- feat: New feature
- fix: Bug fix
- docs: Documentation changes
- style: Code style changes
- refactor: Code refactoring
- test: Adding tests
- chore: Maintenance tasks

## Security

- Never commit sensitive data
- Use environment variables for secrets
- Report security vulnerabilities privately

## Questions?

Create an issue or contact the maintainers.
